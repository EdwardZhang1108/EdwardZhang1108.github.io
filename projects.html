Unix-like kernel from scratch

Designed and implemented a Unix-like kernel supporting multiple virtual address spaces, user-kernel mode transitions, preemptive multitasking, system calls, exception handling, and device-backed console, keyboard, and timer functionality.

  • Built paging-based virtual memory, physical frame allocation, address-space loading, and user pointer validation.

  • Integrated process and thread lifecycle behavior including fork, exec, wait, vanish, thread fork, and fault cleanup.
  • Carried earlier OS components forward: traceback, interrupt-driven drivers, and a user-level thread library.

Distributed Bitcoin mining system

Built a distributed miner that partitions hash search across dynamic workers using a custom Live Sequence Protocol over UDP, fair scheduling, heartbeat detection, retransmission, and failed-work reassignment.

  • Reduced a 9-digit hash search from 28 hours to 17 minutes with 100+ concurrent miners.
  • Handled concurrent clients and miner join/leave behavior without losing in-flight work.

Go · RPC · replicated log

Raft consensus algorithm

Implemented a replicated log with leader election, heartbeats, RequestVote and AppendEntries RPCs, commit-index advancement, and ApplyCommand delivery under delayed, reordered, and dropped RPCs.

  • Maintained progress when a quorum was available and stopped safely without one.
  • Separated protocol state, RPC handling, and committed-entry delivery to avoid deadlocks.

Go · actors · RPC · key-value storage

CMUD backend storage system

Built the backend for a multiplayer text-world service: a geographically distributed, highly available key-value store whose actors answer local queries immediately and synchronize updates in the background.

  • Implemented actor mailboxes, remote tells, query RPCs, and thread-safe KVS client calls.
  • Used last-writer-wins merge semantics for eventual consistency across local and remote actors.

LLM serving · cache memory · empirical evaluation

KV cache compression study

Studied how KV-cache token eviction and value quantization interact under an equal-memory protocol, comparing sequential orderings, MiniKV-style baselines, H2O-style heavy hitters, and a joint scoring strategy.

  • At an 8.39 MB cache budget on Llama-3.2-3B, joint scoring reached 8.62 PPL versus 43.00 / 48.17 for sequential orderings.
  • Used error decomposition to explain when eviction and quantization noise cancel or reinforce.
